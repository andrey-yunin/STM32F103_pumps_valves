# План рефакторинга: Приведение к стандарту DDS-240 (Advanced)

**Цель:** Обеспечить 100% совместимость платы насосов и клапанов с экосистемой DDS-240, включая сервисный слой и отказоустойчивость.

## Этап 1: Архитектурный фундамент (Стандартизация)
- [x] **1.1. Переименование**: Изменение имен файлов и функций `task_command_parser` -> `task_dispatcher`.
- [x] **Синхронизация инфраструктуры**: Переименование очередей в `dispatcher_queueHandle` и `fluidics_queueHandle`.
- [x] **1.2. Модуль конфигурации и идентификации (`app_flash.h/c`)**:
    - [x] Чтение UID чипа (96 бит).
    - [x] Определение Device Type (0x03 - Fluidic) и версии FW.
    - [x] Работа с Page 63 (STM32F103).
    - [x] Структура с Magic Key (0x55AAEEFF) и CRC16.
    - [x] Сохранение/загрузка NodeID и Fluidic Map.
- [x] **1.4. Транспортный уровень (bxCAN)**:
    - [x] Аудит `task_can_handler`: Mailbox Guard (проверка ящиков перед передачей).
    - [x] Поддержка Broadcast (`DstAddr = 0x00`).
    - [x] Переход на динамический NodeID (из `AppConfig`).
    - [x] Реализация `CAN_SendData` с поддержкой EOT.
- [x] **1.5. Архитектурная чистка (Унификация с паттерном)**:
    - [x] Устранение дублирования ACK: Удаление `CAN_SendAck` из `task_pump_controller` (остается только в Диспетчере).
    - [x] Единый источник маппинга: Перевод `device_mapping.c` на работу через `AppConfig`.
    - [x] Консистентность NACK: Замена общих кодов ошибок на специфичные для домена Fluidics.

## Этап 2: Сервисный слой (0xFxxx)
- [x] **2.1. Коды команд**: Синхронизация `can_protocol.h` с эталоном (0xF001-0xF00F, Magic Keys).
- [x] **2.2. Обработка в Диспетчере (`task_dispatcher`)**:
    - [x] `0xF001` (Get Info): Сборка ответа (Type, Ver, UID).
    - [x] `0xF002` (Reboot): NVIC_SystemReset() с проверкой Magic Key `0x55AA`.
    - [x] `0xF003` (Commit): Запись RAM-конфигурации во Flash.
    - [x] `0xF004` (Get UID): Отдельная команда получения серийного номера.
    - [x] `0xF005` (Set NodeID): Динамическое изменение адреса в RAM.
    - [x] `0xF006` (Factory Reset): Сброс настроек и перезагрузка (Key: 0xDEAD).
